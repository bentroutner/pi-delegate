### Design Decisions

**File Transfer Cleanup**
- Used `try/finally` to guarantee cleanup even on failure
- Unique temp paths with timestamps to avoid collisions
- Cleanup failures log warning but don't fail the operation
- Critical for jump server's 11GB limit

**Pi Communication**
- SSH hop through jump server for all Pi commands
- tmux load-buffer/paste-buffer pattern for safe JSON delivery
- No manual JSON escaping needed

**Polling Strategy**
- Simple completion detection based on turn_end markers
- Error detection from Pi's JSON responses
- Configurable interval and max attempts

### Testing & Debugging

**First Test Failure**
- Pi not responding to messages
- Root cause: JSON not properly escaped through SSH hops
- Error: `Failed to parse command: Expected property name or '}' in JSON`

**Fix 1: Base64 Encoding**
- Instead of shell escaping, encode JSON as base64
- Decode on Pi with `base64 -d`
- Eliminates all quote/escape issues

**Fix 2: Newline Handling**
- `echo` adds newlines that break JSON
- Switched to `printf %s` for raw output

**Fix 3: Message Submission**
- Pi needs Enter keypress after pasting JSON
- Added `tmux send-keys Enter` after paste

**Fix 4: Error Detection**
- Initial logic flagged tool errors as failures
- Pi often runs bash commands that fail (normal behavior)
- Changed to only detect system errors, not tool errors

**Successful Test**
- Task file transferred successfully
- Pi received and processed message
- Pi created hello.js following all style conventions
- Completion detected correctly
- Output verified on Pi server

### Lessons Learned

1. **SSH JSON transmission is hard** — Base64 encoding is the reliable solution
2. **Pi needs Enter key** — Pasting isn't enough, must submit with Enter
3. **Tool errors are normal** — Don't panic when Pi's bash commands fail
4. **3-minute polling works** — Gives Pi time to work without interruption
5. **Jump server cleanup is critical** — 11GB fills up fast without cleanup
